## A. Спасти принцессу

**Ограничение времени:** 2 с  
**Ограничение памяти:** 256 МБ  
**Ввод:** стандартный ввод или `input.txt`  
**Вывод:** стандартный вывод или `output.txt`

### Условие

Вы играете в компьютерную игру и управляете героем. Его цель — спасти принцессу, спрятанную в одной из пещер подземелья.

Подземелье состоит из **пещер** и **проходов** между ними. Схема подземелья образует **дерево**: между любыми двумя пещерами существует ровно один путь.

Герой находится в пещере `s`, а принцесса — в пещере `t`.

В каждом проходе сидит монстр. Для прохода задано целое число `w` — уровень опасности монстра. Если герой проходит по этому проходу, то вероятность победить монстра равна:

```
p = 1 / exp(w)
```

Герой идёт по единственному пути из `s` в `t` и должен победить всех монстров на его пути.

Итоговая вероятность добраться до принцессы:

```
P = 1 / exp(W)
```

**Ваша задача** — вычислить и вывести целое число `W`.

### Формат ввода

В первой строке заданы три целых числа:
- `n` (2 ≤ n ≤ 10⁵) — число пещер
- `s` (1 ≤ s ≤ n) — пещера героя
- `t` (1 ≤ t ≤ n) — пещера принцессы

Далее следует `n − 1` строка с описанием проходов. Каждая строка содержит три числа: `a_i`, `b_i` (номера пещер) и `w_i` (1 ≤ w_i ≤ 10⁹) — уровень опасности монстра в проходе.

### Формат вывода

Выведите целое число `W`.

### Примеры

**Пример 1:**
```
Ввод:        Вывод:
2 1 2        1
1 2 1
```

**Пример 2:**
```
Ввод:        Вывод:
5 3 2        7
1 2 6
3 5 2
1 4 5
1 3 1
```

## B. Древний артефакт

**Ограничение времени:** 3 с  
**Ограничение памяти:** 512 МБ  
**Ввод:** стандартный ввод или `input.txt`  
**Вывод:** стандартный вывод или `output.txt`

### Условие

Вася учится в университете на археолога. Во время практики он нашёл шар, на котором написано **целое неотрицательное число длины N**. На свитке рядом с шаром сказано, что древняя цивилизация поклонялась числам, кратным 75.

**Задача:** найти минимальное количество цифр, которые нужно удалить из написанного на шаре числа, чтобы оставшееся число было кратно 75.

> **Примечание 1.** Если удалить все цифры, число становится равным нулю и делится на 75.  
> **Примечание 2.** Если удалить все цифры, кроме нулей, число также равно нулю.

### Формат ввода

Дано целое неотрицательное число длины N (1 ≤ N ≤ 10⁶).

### Формат вывода

Выведите число `k` — минимальное количество цифр, которое нужно удалить.

### Примеры

| Ввод    | Вывод | Пояснение |
|---------|-------|-----------|
| `17523` | `3`   | Удаляем цифры 1, 2, 3 → получаем 75 |
| `111`   | `3`   | Удаляем все цифры → получаем 0 |
| `1000`  | `1`   | Удаляем единственную ненулевую цифру → получаем «000» = 0 |

## C. На пересечении проспекта и бульвара

### Условие

В городе две главные улицы пересекаются под прямым углом у станции метро (точка O):
- **Проспект** — идёт строго с запада на восток
- **Бульвар** — идёт строго с юга на север

**Маршрут курьера:**
- Старт: точка **A** на Проспекте, 1800 м к **западу** от метро
- Обязательная остановка: офис **V** — 160 м **восточнее** и 150 м **севернее** метро (на открытой территории)
- Финиш: постамат **B** на Бульваре, 2100 м к **северу** от метро

**Скорости движения до посещения офиса (обычный шаг):**
- По Проспекту: 1.7 м/с
- По Бульвару: 2.0 м/с
- По открытой территории: 1.2 м/с

**Скорости движения после посещения офиса (лёгкий бег):**
- По любой улице: 2.3 м/с
- По открытой территории: 1.2 м/с

**Задача:** найти минимальное время (в секундах), за которое курьер, обязательно посетив точку V, может добраться из A в B.

### Формат вывода

Минимальное время в секундах, округлить до сотых.

Пример формата: `123.45`

## D. TwoFeaturesGame

**Ограничение времени:** 60 с  
**Ограничение памяти:** 1024 МБ  
**Ввод:** `test_without_target.csv`  
**Вывод:** `test_target_only.csv`

### Условие

В мультиплеерной игре вышел патч с серьёзными изменениями баланса. Игроков нужно разделить на опытных и неопытных (метки 1 и 2, в равном соотношении) для матчмейкинга.

Аналитики выделили два признака: `feature_1` и `feature_2`. Модель обучили на старых данных (`train_with_target.csv`), но после патча распределение признаков изменилось и старая модель перестала работать.

**Задача:** предсказать метки (1 или 2) для игроков из тестовой выборки (`test_without_target.csv`) без новой разметки. Аналитики подтвердили, что **структурная связь между признаками сохранилась**.

### Формат вывода

Файл `test_target_only.csv`:
- Первая строка: `target`
- Далее: метки 1 или 2 в том же порядке, что и в тестовых данных

Замена всех 1 на 2 и наоборот не меняет корректность.

### Система оценивания

Модель считается успешной, если `|Accuracy − 1/2| ≥ √30/20 ≈ 0.27`.

Гарантируется, что в тестовой выборке одинаковое число игроков каждого класса.

## E. Утерянный язык

**Ограничение времени:** 60 с  
**Ограничение памяти:** 64 МБ  
**Ввод:** `text.txt`  
**Вывод:** `submission.json`

### Условие

Во время экспедиции найдена книга на неизвестном языке. Исследователям удалось расшифровать ограниченное число фрагментов — пары (оригинал, перевод на русский).

**Задача:** восстановить переводы оставшихся фрагментов.

Требование: набрать **не ниже 20 баллов по метрике BLEU** на тестовой выборке.

Разрешается дообучить любую open source модель (например, T5-small).

### Формат ввода

- `train_data.json` — пары переводов для обучения
- `test_data.json` — фрагменты для перевода

### Формат вывода

Файл `submission.json` (код должен содержать готовые ответы, а не инферить модель):

```json
[
  { "source": "JCljõft ff jí lpnãóéC", "translation": "..." },
  { "source": "ôẑkà pṛïmõššëd lüǵëñ", "translation": "..." }
]
```

## F. Заклинание продолжения

**Ввод:** `input.txt`  
**Вывод:** `output.txt`

### Условие

Гриша спрятал пароли с помощью языковой модели. Для каждого пароля он придумал «заклинание-префикс» (passphrase) и отправил его модели:

```python
messages = [
    {"role": "user", "content": f"Just generate a continuation of the phrase: {passphrase}"}
]
```

То, что модель генерирует в ответ — и есть пароль.

Гриша дообучил модель **HuggingFaceTB/SmolLM2-360M-Instruct** на своих парах (passphrase → пароль), но потерял гримуар с паролями. Остались только passphrase и доступ к модели.

**Известно о паролях:**
- Ровно 16 символов
- Только цифры и латинские буквы (a–z, A–Z)
- Каждый символ пароля = ровно один токен

**Задача:** восстановить все пароли из `input.txt`.

### Формат вывода

Файл `output.txt` — по одному паролю на строку, в том же порядке, что и passphrase во входных данных.

> При инициализации модели использовать `dtype="auto"`.
